# VS Code 自动保存配置指南

在编写代码时，开启自动保存（Auto Save）功能可以有效避免因意外断电、软件崩溃或忘记保存而导致的代码丢失。VS Code 提供了多种灵活的自动保存策略，本文将介绍如何配置和使用该功能。

## 方式一：通过菜单栏快速开启（最简单）

如果你只需要最基本的自动保存功能，可以通过以下操作快速开启：

1. 点击顶部菜单栏的 **文件 (File)**。
2. 在下拉菜单中找到并勾选 **自动保存 (Auto Save)**。

> **提示**：启用后，VS Code 默认会在你每次修改代码后应用延时保存。

## 方式二：通过设置界面进行详细配置

如果你希望更精细地控制保存的时机，可以通过设置面板进行修改：

1. 打开设置面板，选择以下任一方式：
   - 点击左下角的齿轮图标，然后选择 **设置 (Settings)**。
   - Windows / Linux: 按下 `Ctrl + ,`
   - macOS: 按下 `Cmd + ,`
   - 或者通过菜单栏：**文件 (File) -> 首选项 (Preferences) -> 设置 (Settings)**。
2. 在顶部的搜索框中输入 `Auto Save`。
3. 找到 **Files: Auto Save** 选项。你会看到一个下拉菜单，里面有四个选项：
   - `off`：关闭自动保存（默认值），你需要手动按 `Ctrl + S` 保存。
   - `afterDelay`：延迟保存。在停止输入经过指定的时间延迟后自动保存。
   - `onFocusChange`：焦点改变时保存。当你的光标离开当前正在编辑的文件（例如切换到另一个文件标签或侧边栏）时，自动保存该文件。
   - `onWindowChange`：窗口切换时保存。当你将焦点从 VS Code 窗口切换到其他应用程序（例如去浏览器查资料）时进行保存。

如果选择了 `afterDelay`，你还可以配置 **Files: Auto Save Delay** 来设置延迟的时间，单位为毫秒（默认是 `1000` 也就是 1 秒）。

## 方式三：通过 `settings.json` 进阶配置

喜欢直接编辑配置文件的开发者，可以直接修改全局或当前工作区的 `settings.json` 文件：

1. 按 `Ctrl + Shift + P`（macOS: `Cmd + Shift + P`）打开命令面板。
2. 输入 `Open Settings (JSON)` 并回车打开全局配置文件，或者输入 `Open Workspace Settings (JSON)` 打开当前项目的配置文件。
3. 在 JSON 文件中添加或修改以下键值：

```json
{
  // 设置自动保存模式："off", "afterDelay", "onFocusChange", "onWindowChange"
  "files.autoSave": "afterDelay",

  // 设置延迟时间（仅在 "afterDelay" 模式下生效），单位毫秒
  "files.autoSaveDelay": 1000
}
```

## 不同自动保存模式的适用场景

在实际开发中，不同的保存模式适合不同的工作流：

- **`afterDelay`（延迟保存）**：适合不习惯手动保存且不依赖“保存时热更新”的项目。但是如果你配置了“保存时自动格式化 (Format On Save)”或者前端的热更新模块（如 Webpack/Vite），不建议使用此模式，否则打字略有停顿就会引发频繁的代码格式闪烁或页面刷新。
- **推荐策略**：推荐使用 **`onFocusChange`** 甚至是 **`onWindowChange`**。这样既能保证在你切换去其他文件或去浏览器查阅资料时进度得到保存，又避免了刚写一半代码就被工具强行格式化的困扰。
